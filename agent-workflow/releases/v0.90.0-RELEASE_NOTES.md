## Summary

Dashboard primary-workspace Phase A (action strip, project quick drawer, return-to-dashboard) behind `VITE_FEATURE_DASHBOARD_WORKSPACE`, plus collapsed-shell persistence and a hover-only compact icon rail that expands the full sidebar only on FAB click. Project entry sticky header and role tabs stack without overlap.

## Features added

- Dashboard “Do next” action strip (menu-gated New Project / Overdue / Worklist / Support / CNF)
- Project quick drawer from worklist, notifications, Due Soon, and Recent Edit (Final Status edit for PP/Admin)
- Drill routes append `return_to=/dashboard`; filter banner “Back to Dashboard”
- Feature kill-switch `VITE_FEATURE_DASHBOARD_WORKSPACE` + `DASHBOARD_WORKSPACE_ROLLBACK.md`
- Collapsed chrome stays collapsed across navigation (session memory + `sessionStorage`)
- Hover expand FAB → compact icon rail with tooltips; click FAB → full sidebar + topbar
- Project sticky stack: title + role tabs pin together without overlap

## Defects fixed

- Sticky role tabs overlapping project sticky header while scrolling
- Collapsed sidebar/topbar resetting to expanded on every route change (AppShell remount)

## Security / configuration changes

- Optional build flag `VITE_FEATURE_DASHBOARD_WORKSPACE` (default on). No new schema/migration.
- Quick drawer uses existing `updateProject` + menu/role gates; Meeting View remains read-only.

**Risk:** Low–Medium. Rollback via flag `false` + redeploy, or redeploy `0.89.0` @ `82bc127`.
