## Summary

Projects Database uses spreadsheet-style blank draft rows (no Add project bar) with bulk create on Save; app Back/Forward restores priority page UI state; workflow status columns show compact icons with tooltips.

## Features / UX

- Fill-viewport blank editable draft rows; typing in the last blank appends another
- Save creates one project per non-blank valid draft (`saveProject` / unique `PROJ-YYYY-NNN`); patches existing edits; skips blank rows; blocks on cell validation errors
- Compact workflow status icons in Projects Database (filter/sort/edit keep full text)
- Back/Forward navigation + view-state restore (Worklist, Project Entry, Projects DB, Support, CNF)
- Worklist search placement, Project Owner column, Process open icon
- Project Entry hierarchy expanded by default on load/create
- Sidebar session clear on same-tab user switch; creatable cell editor mousedown fix

## Security / configuration changes
